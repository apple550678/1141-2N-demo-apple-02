React 入門比較指南：Create React App vs Vite

專案來源：w01-create-react-app 與 w01-vite-app

學習目標

  • 理解兩種 React 起手式：Create React App（CRA）與 Vite 的差異。
  • 看懂入口檔與掛載流程，能自行修改畫面。
  • 能啟動、建置專案，並知道何時該選哪一種工具。

快速結論

CRA（react-scripts + Webpack）

  • 指令：npm start、npm run build、npm test
  • 特點：一站式模板，含測試設定、reportWebVitals。
  • 開發啟動較慢，配置隱藏，需 eject 才能微調。

Vite（原生 ESM + Rollup）

  • 指令：npm run dev、npm run build、npm run preview
  • 特點：冷啟與 HMR 很快，index.html 是一級資產。
  • 配置清楚、可擴充，現代前端常用的預設。

專案結構與關鍵檔案

CRA：w01-create-react-app

  • src/index.js：入口與掛載（ReactDOM.createRoot(document.getElementById('root'))）。
  • src/App.js：主要畫面元件；顯示 logo、連結與一段文字。
  • public/index.html：模板檔；建置時由打包工具注入 bundle。
  • src/reportWebVitals.js：可記錄效能指標（非必須）。
  • src/App.test.js、src/setupTests.js：測試樣板。

Vite：w01-vite-app

  • index.html：真實載入的 HTML，以 <script type="module" src="/src/main.jsx"> 引入入口檔。
  • src/main.jsx：掛載 React 應用（createRoot(...).render(...)）。
  • src/App.jsx：包含 useState 的互動計數器與示例樣式。
  • vite.config.js：啟用 React 外掛與 Vite 設定。

程式碼運作流程（核心）

  1. 瀏覽器載入 HTML，取得 <div id="root"> 容器。
  2. 入口檔（CRA：src/index.js；Vite：src/main.jsx）初始化 React Root。
  3. 以 <StrictMode> 包覆 <App/>，協助在開發模式偵錯。
  4. App 元件回傳 JSX，組成畫面；Vite 範例中包含一個計數器狀態。

如何啟動與建置

CRA

cd demo/w01-create-react-app
npm start       # 開發伺服器（http://localhost:3000）
npm run build   # 產出 production 檔案

Vite

cd demo/w01-vite-app
npm run dev     # 開發伺服器（通常是 http://localhost:5173）
npm run build   # 打包
npm run preview # 在本地預覽 build 結果

差異重點與選用建議

  • 啟動速度：Vite 明顯較快（原生 ESM、按需載入）。
  • 模板定位：CRA 使用 public/index.html 模板；Vite 的 index.html 即是入口。
  • 生態與擴充：Vite 擴充容易、配置直觀；CRA 偏向封裝（需 eject 才能深度客製）。
  • 測試：CRA 預設帶有測試樣板；Vite 可自行整合 Vitest/Jest。
  • 學習路徑：初學者都可；若追求流暢開發體驗與現代化工具，建議選 Vite。

常見問題：亂碼（「?�?…213410102」）

在 App.js 與 App.jsx 中皆看到疑似亂碼的字串，可能是存檔編碼非 UTF-8 或字型顯示所致。

  1. 確認編輯器檔案編碼為 UTF-8 後重存。
  2. 以中文（繁體）直接輸入，或使用 Unicode 轉義。
  3. 瀏覽器端幾乎都以 UTF-8 處理，主要來源是檔案編碼。

延伸練習

  • 在兩個專案的 App 中各新增一個按鈕，切換深色主題。
  • 將共同的頁面文案抽出成一個 Message 元件，於兩邊共用。
  • 嘗試在 Vite 專案加入 react-router，做兩頁導覽。
